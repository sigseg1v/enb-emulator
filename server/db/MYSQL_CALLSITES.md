# MySQL call site survey (Phase C)

## Raw `mysql_*` C API usage

Only 3 files touch the raw libmysqlclient C API:

- `server/src/mysql/mysql.h`       — vendored mysql header (will be removed)
- `server/src/mysql/mysqlplus.h`   — wrapper class declarations
- `server/src/mysql/mysqlplus.cpp` — wrapper class implementations

Net-7 already abstracted DB access behind the `sql_connection_c` / `sql_query_c` / `sql_result_c` classes. The migration shape is therefore **rewrite one file (mysqlplus.cpp) against libpqxx**, and the rest of the server tracks transparently.

Each of these files holds SQL strings or query-result accessors but does **not** call MySQL C functions. SQL dialect compatibility is the remaining concern (Phase C continuation):

- LIMIT syntax: identical between MySQL and Postgres.
- AUTO_INCREMENT inserts: net7's INSERT statements use explicit IDs or omit the column; `GENERATED BY DEFAULT AS IDENTITY` (per decisions log 2026-05-22) accepts both.
- `\`backticks\``: callers must use double-quotes or unquoted identifiers — grep them.
- `INSERT ... ON DUPLICATE KEY UPDATE`: needs rewrite to `ON CONFLICT (..) DO UPDATE`.
- `LAST_INSERT_ID()`: needs rewrite to `RETURNING id`.

Grep counts (Phase C continuation will work down these):

```
$ grep -c "ON DUPLICATE KEY UPDATE" server/src/*.cpp server/src/*.h | grep -v ':0$'
$ grep -c "LAST_INSERT_ID" server/src/*.cpp server/src/*.h | grep -v ':0$'
$ grep -c 'INSERT.*\\\`'         server/src/*.cpp server/src/*.h | grep -v ':0$'
```
